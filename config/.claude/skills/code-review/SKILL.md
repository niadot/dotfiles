# Code Review スキル

ファイルやディレクトリのコードレビューを実施します。

## 説明
指定されたコードに対して、品質、セキュリティ、パフォーマンス、ベストプラクティスの観点からレビューを行います。

## 使い方
```
/code-review <ファイルパスまたはディレクトリ>
```

## 指示

ユーザーがこのスキルを実行したとき、以下の手順を実施してください：

1. **ファイル/ディレクトリの確認**
   - 指定されたパスのファイルを読み取る
   - ディレクトリの場合は主要なファイルを特定する

2. **コードレビューの実施**
   以下の観点で分析してください：
   - **バグや潜在的な問題**: ロジックエラー、エッジケースの未処理など
   - **セキュリティ**: XSS、SQLインジェクション、認証・認可の問題など
   - **パフォーマンス**: 非効率なループ、不要な計算、メモリリークなど
   - **可読性**: 命名規則、コメント、コードの構造
   - **ベストプラクティス**: 言語/フレームワーク固有の推奨事項

3. **レポートの作成**
   以下の形式で結果を報告してください：

   ```
   ## コードレビュー結果

   ### ✅ 良い点
   - [具体的な良い点を列挙]

   ### ⚠️ 改善が推奨される点
   - [優先度: 高/中/低] 問題の説明
     - 理由: ...
     - 推奨される修正: ...

   ### 📊 総合評価
   - 全体的な評価とコメント
   ```

4. **フォローアップ**
   - 修正が必要な場合は、修正を実施するか提案する
